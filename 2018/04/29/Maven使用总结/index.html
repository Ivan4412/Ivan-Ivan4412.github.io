Maven使用总结

  2018-04-29    Spring    SpringBoot  Maven

1. Maven是什么？

Maven是一个跨平台的项目管理工具，它提供了简化和标准化的项目建设过程（构建管理，依赖管理，文档生成及综合项目信息管理。

Maven的特点：

  1. 依赖管理。Maven自动下载所需jar，并统一管理项目中Jar包的依赖关系。
  2. 约定优于配置。约定java源码、资源、测试代码、编译代码存放位置。
  3. 生命周期管理，自动构建。例mvn install就可以自动执行编译，测试，打包等构建过程。

2. Maven目录约定

MavenProjectRoot(项目根目录)   
   |----src  
   |     |----main  
   |     |         |----java ——存放项目的.java文件  
   |     |         |----resources ——存放项目资源文件，如spring, hibernate配置文件  
   |     |----test
   |     |         |----java ——存放所有测试.java文件，如JUnit测试类  
   |     |         |----resources ——存放项目资源文件，如spring, hibernate配置文件  
   |----target ——项目输出位置  
   |----pom.xml ----用于标识该项目是一个Maven项目

3.POM文件

POM文件定义了项目的一些基本信息（如，项目坐标/依赖配置/构建配置/项目信息/环境配置）

<project xmlns="http://maven.apache.org/POM/4.0.0"
     xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
     xsi:schemaLocation="http://maven.apache.org/POM/4.0.0
        http://maven.apache.org/xsd/maven-4.0.0.xsd">
<modelVersion>4.0.0</modelVersion> <!-- pom模型版本，maven2和3只能为4.0.0 -->

<!-- 父项目基本信息 -->
<parent>
    <groupId>com.rongdu</groupId>  <!-- 父项目组ID -->
    <artifactId>p2psys</artifactId> <!--父项目组中的唯一ID-->
    <version>2.0.0.0</version>  <!-- 父项目版本 -->
    <relativePath>../p2psys/pom.xml</relativePath> <!-- 父项目的pom文件存放相对位置 -->
</parent>

<!--  子模块配置 -->
<modules>
    <module>../p2psys-core</module>
    <module>../p2psys-manage</module>
</modules>

<!-- 基本配置 -->
<groupId>com.rongdu.p2psys</groupId>  <!-- 组ID -->
<artifactId>p2psys-core-bank</artifactId> <!--在组中的唯一ID，一般为项目名或模块名 -->
<version>2.0.0.0</version>  <!-- 项目版本 -->
<packaging>jar</packaging> <!-- 项目打包方式，有以下值：pom, jar, maven-plugin, ejb, war, ear, rar, par -->

<!-- 定义变量 -->
<properties>
    <junit.version>1.7</junit.version>
    <project.build.sourceEncoding>UTF-8</project.build.sourceEncoding>
</properties>


<!-- 依赖配置 -->
<dependencies>
    <dependency>
        <groupId>junit</groupId>
        <artifactId>junit</artifactId>
        <version>${junit.version}</version>
        <scope>test</scope> <--依赖生命周期管理方式，值可为：compile(默认值)、test、runtime、provided、system -->
    </dependency>
</dependencies>

<!--  父项目依赖配置方式；好处是，父项目统一版本，子项目在需要的时候引用 -->
<dependencyManagement>
    <dependencies>
        <dependency>
            <groupId>junit</groupId>
            <artifactId>junit</artifactId>
            <version>4.12</version>
            <scope>test</scope>
            <type></type> <!--依赖类型，对应项目坐标定义的packaging，默认不用声明，为jar -->
            <optional></optional> <!-- 标记依赖是否可选-->
            <exclusions> <!-- 排除传递性依赖 -->
                <exclusion>
                    <artifactId></artifactId>
                    <groupId></groupId>
                </exclusion>
            </exclusions>
        </dependency>
    </dependencies>
</dependencyManagement>

<!--  对应项目依赖配置方式 -->
<dependency>
    <groupId>junit</groupId>
    <artifactId>junit</artifactId>
</dependency>

<!-- 构建配置 -->
<build>
    <directory/>  <!--target目录-->
    <outputDirectory/> <!--被编译过的应用程序class文件存放的目录。-->  
    <finalName/> <!--产生的构件的文件名--> 
    <testOutputDirectory/> <!--被编译过的测试class文件存放的目录。-->
    <sourceDirectory/>  <!--项目源码目录，构建系统会编译目录里的源码，相对于pom.xml的相对路径 -->     
    <scriptSourceDirectory/>   <!--项目脚本源码目录，该目录下的内容，会被拷贝到输出目录(因为脚本是被解释的，而不是被编译的)。-->    
    <testSourceDirectory/>   <!--项目单元测试使用的源码目录-->    

    <!--资源路径-->
    <resources>     
        <resource>    
        <targetPath/>    <!-- 资源的目标路径。相对target/classes目录-->          
        <directory/>    <!--描述存放资源的目录，该路径相对POM路径-->        
        <includes/>    <!--包含的模式列表，例如**/*.xml.-->    
        <excludes/>    <!--排除的模式列表，例如**/*.xml-->    
      </resource>    
    </resources> 

    <testResources>
      <testResource>
        <directory>${project.basedir}/src/test/resources</directory>
      </testResource>
    </testResources>

    <!--使用的插件列表--> 
    <pluginManagement>
       <plugins>
        <plugin>
            <groupId>org.apache.maven.plugins</groupId>
            <artifactId>maven-jar-plugin</artifactId>
            <version>3.0.2</version>
            <configuration>
            <excludes>
                <exclude>application.yml</exclude>
            </excludes>
        </configuration>
        </plugin>
    </plugins>
    </pluginManagement>

   <!--外部构建扩展-->    
    <extensions>    
       <extension>       
        <groupId/>     <!--构建扩展的groupId-->    
        <artifactId/>    <!--构建扩展的artifactId-->      
        <version/>      <!--构建扩展的版本-->    
       </extension>    
    </extensions>

</build>    

<!--产生报表的规范-->
<reporting>...</reporting>

<!-- 项目信息 -->
<name>...</name>
<description>...</description>
<url>...</url>
<inceptionYear>...</inceptionYear>
<licenses>...</licenses>
<organization>...</organization>
<developers>...</developers>
<contributors>...</contributors>

<!-- 环境设置 -->
<issueManagement>...</issueManagement>
<ciManagement>...</ciManagement>
<mailingLists>...</mailingLists>
<scm>...</scm>
<prerequisites>...</prerequisites>
<repositories>...</repositories>
<pluginRepositories>...</pluginRepositories>
<distributionManagement>...</distributionManagement>
<profiles>...</profiles>

</project>

4. Maven的scope属性

  1. compile：默认值，依赖项目需要参与当前项目的编译，测试，运行周期，打包时需要包含进去。
  2. test：仅参与测试的编译和执行，不会被打包
  3. runtime：无需参与项目的编译，但参与后期的测试和运行周期。相比compile，跳过了编译。
  4. provided：打包的时候可以不用包进去。相比compile，打包阶段做了exclude操作。
  5. system：从参与度来说，和provided相同，不过被依赖项不会从maven仓库下载，而是从本地文件系统拿。需要添加systemPath的属性来定义路径。

5. 可选依赖

该依赖只对当前引用有效，不会向下传导。
如果A依赖于B，B中有如下依赖

<dependency>
    <groupId>com.my.C</groupId>
    <artifactId>cpt</artifactId>
    <version>1.0.0</version>
    <optional>true</optional>
</dependency>

通过上面配置，optional为true，那么A中就不会依赖这个cpt

6. 排除依赖

如果A–>B,B–>C1，通过排除依赖，A排除对C1的依赖，可替换成C2。
A的依赖关系如下配置：

<dependency>
    <groupId>com.yjs.b</groupId>
    <artifactId>pro-b</artifactId>
    <version>1.0.1</version>
    <!--排除依赖-->
    <exclusions>
        <exclusion>
            <groupId>com.yjs.c1</groupId>
            <artifactId>pro-c1</artifactId>
        </exclusion>
    </exclusions>
</dependency>
<!-- 引入正确依赖 -->
<dependency>
    <groupId>com.yjs.c2</groupId>
    <artifactId>pro-c2</artifactId>
    <version>1.0.0</version>
</dependency>

7. Maven常用命令

Maven常用命令：

  1. 创建Maven的普通java项目：

    mvn archetype:create 
-DgroupId=packageName 
-DartifactId=projectName

  2. 创建Maven的Web项目：

    mvn archetype:create 
-DgroupId=packageName    
-DartifactId=webappName 
-DarchetypeArtifactId=maven-archetype-webapp
  3. 编译源代码： mvn compile
  4. 编译测试代码：mvn test-compile
  5. 运行测试：mvn test
  6. 产生site：mvn site
  7. 打包：mvn package
  8. 在本地Repository中安装jar：mvn install
  9. 清除产生的项目：mvn clean
  10. 生成eclipse项目：mvn eclipse:eclipse
  11. 生成idea项目：mvn idea:idea
  12. 组合使用goal命令，如只打包不测试：mvn -Dtest package
  13. 编译测试的内容：mvn test-compile
  14. 只打jar包: mvn jar:jar
  15. 只测试而不编译，也不测试编译：mvn test -skipping compile -skipping test-compile
  16. 清除eclipse的一些系统设置:mvn eclipse:clean
